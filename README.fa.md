# چت‌بات پرسش و پاسخ اسناد PDF بر پایه RAG

این مخزن شامل یک پروژه کامل و آماده برای پرتفوی است که امکان بارگذاری فایل‌های PDF و پرسش درباره محتوای آن‌ها را فراهم می‌کند. این چت‌بات متن اسناد را استخراج، آن را به بخش‌های کوچک تقسیم و با استفاده از جستجوی برداری، بخش‌های مرتبط را بازیابی می‌کند و پاسخی مبتنی بر منابع ارائه می‌دهد. این پروژه برای یادگیری و استفاده در مصاحبه‌های فنی طراحی شده است.

> **لطفاً توجه کنید:** نسخهٔ انگلیسی راهنما در فایل [`README.md`](README.md) موجود است. این فایل اطلاعات را به زبان فارسی ارائه می‌کند تا کاربران فارسی‌زبان نیز بتوانند از پروژه بهره ببرند.

## معرفی

در این پروژه یک خط لولهٔ «بازیابی افزوده به تولید» (RAG) پیاده‌سازی شده است. کاربران می‌توانند از طریق رابط وب ساخته شده با **Streamlit** یا از طریق خط فرمان با سیستم تعامل کنند. مراحل اصلی عبارتند از:

1. **بارگذاری PDF** – بارگذاری یک یا چند فایل PDF.
2. **استخراج متن** – استخراج متن از هر صفحه با کتابخانهٔ `pypdf` و ذخیرهٔ نام سند و شمارهٔ صفحه.
3. **تقسیم متن** – تقسیم متن هر صفحه به بخش‌های همپوشان (به طور پیش‌فرض طول هر بخش ۸۰۰ کاراکتر و همپوشانی ۱۵۰ کاراکتر است).
4. **برداردهی** – تبدیل هر بخش به یک بردار عددی.  به طور پیش‌فرض از یک مدل معنایی پیش‌آموزش دیدهٔ SentenceTransformer ("all‑MiniLM‑L6‑v2") استفاده می‌شود که علاوه بر مطابقت واژگانی، معنا را نیز در نظر می‌گیرد.  اگر این مدل یا کتابخانهٔ مربوطه در دسترس نباشد، از مدل سبک‌تر TF‑IDF به عنوان گزینهٔ پشتیبان استفاده می‌شود.
5. **ذخیرهٔ برداری** – ذخیرهٔ بردارها و اطلاعات متا در یک فروشگاه برداری محلی.
6. **بازیابی** – با دریافت پرسش کاربر، بخش‌های مرتبط بازیابی می‌شوند.
7. **تولید پاسخ** – پاسخ مختصر و مبتنی بر منابع تولید و به همراه بخش‌های مرجع (نام فایل و شمارهٔ صفحه) نمایش داده می‌شود. اگر اطلاعات کافی وجود نداشته باشد، پیام مناسبی نمایش داده می‌شود.

## نصب روی ویندوز ۱۱

1. مخزن را کلون کنید و وارد پوشه شوید:

   ```bash
   git clone https://github.com/Erfan-Moshfeghi/pdf-question-answering-rag-chatbot.git
   cd pdf-question-answering-rag-chatbot
   ```

2. یک محیط مجازی ایجاد و فعال کنید:

   ```bash
   python -m venv .venv
   .\.venv\Scripts\activate
   ```

3. وابستگی‌ها را نصب کنید:

   ```bash
   pip install -r requirements.txt
   ```

پس از نصب وابستگی‌ها، اجرای اولیه با مدل معنایی ممکن است نیاز به دانلود حدود ۹۰ مگابایت وزن مدل داشته باشد. این دانلود توسط کتابخانهٔ `sentence-transformers` انجام می‌شود و به اتصال اینترنت نیاز دارد. اگر دانلود موفق نشود یا این کتابخانه نصب نباشد، برنامه به طور خودکار از مدل سادهٔ TF‑IDF استفاده می‌کند.

## اجرای برنامهٔ Streamlit

برای اجرای رابط وب:

```bash
streamlit run app.py
```

صفحه‌ای در مرورگر باز می‌شود که می‌توانید فایل‌های PDF را بارگذاری، تنظیمات بخش‌بندی (chunk size و overlap) را تغییر و سوالات خود را مطرح کنید.

## اجرای خط فرمان

برای پردازش یک PDF و پرسیدن یک سوال از طریق CLI:

```bash
python main.py --pdf مسیر/به/فایل.pdf --question "سؤال شما"
```

پاسخ و بخش‌های مرجع در خط فرمان نمایش داده می‌شود.

## تست‌ها

برای اطمینان از صحت عملکرد کد، تست‌هایی نوشته شده‌اند. پس از نصب وابستگی‌ها، از دستور زیر استفاده کنید:

```bash
python -m pytest
```

## محدودیت‌ها

* **PDFهای اسکن شده**: این پروژه به طور پیش‌فرض OCR انجام نمی‌دهد؛ بنابراین PDFهای اسکن شده قابل پردازش نیستند.
* **مدل‌های برداری**: به طور پیش‌فرض از مدل معنایی SentenceTransformer استفاده می‌شود. در صورت نبود امکان بارگذاری این مدل، سامانه به‌طور خودکار از TF‑IDF استفاده می‌کند که فقط بر اساس فراوانی کلمات عمل می‌کند و ممکن است مترادف‌ها یا پارافریز‌ها را تشخیص ندهد.
* **پاسخ‌های استخراجی**: پاسخ‌ها با ترکیب بخش‌های بازیابی شده ساخته می‌شوند و از مدل زبانی بزرگ استفاده نمی‌شود.
* **ذخیره‌سازی محلی**: داده‌ها در حافظه و فایل‌های محلی نگهداری می‌شوند و در جلسات مختلف باقی نمی‌مانند.
